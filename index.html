Intro

JavaScript is an object oriented programming language. It is a very popular language that is primarily used to make pages more interactive. JavaScript is cross-platform and can be used client side or server side. Anyone can learn JavaScript. This light introduction to JavaScript aims to help you better understand the language. A few of JavaScript's notable features are seen below.

Features

  • Dynamic Typing
  • Functional Style
  • Object-oriented
  • Aysnc Processing
  • Client/Server Side
  • Cross-platform
  • Prototype Based
Numbers

JavaScript numbers are always stored as double precision, 64-bit, IEEE 754 floating point values. There are no type definitions for float, integer, short, long etc. Numbers can be written with or without decimals.

x = 42

y = 3.14

Integers can be written with up to 15 digits places with accuracy. 17 decimals is the maximum number allowed. Floating point arithmetic can be inaccurate at times so be mindful. Numbers stored in strings can be converted to integers using the built-in parseInt() function. The function parseFloat() can be used to convert floating point numbers.

There is a special value, NaN, which stands for "Not a Number".

parseInt("Hi") // NaN

Strings

Known as sequences of Unicode characters. They are used for storing and manipulating text. Strings have values encased in single or double quotes. Quotes can be used inside the string as long as they do not match the outside quotes encasing the string.

message = "It's possible to have a single quote inside of a string with double quotes."

message1 = 'This is "also" possible.'

The length of a string can be found using the length property.

message.length // 24

Other Types

Booleans

Booleans have one of two values, true or false. The Boolean() function can be used to test if an expression is true.

Boolean(9 < 12) // returns True

    Boolean() Rules
  1. false, 0, empty strings, NaN, null and undefined are false
  2. All other values are true.

Null

Null is a value that deliberately specified as a non-value.

x = null
console.log(x) // null

Undefined

undefined specifies that a value has not been initialized. Undefined is a constant.

let x
console.log(x) // undefined

Variables

Variables are things that store values.

They can be declared in JavaScript with the var, let,and const keywords. Var declarations can be accessed in the whole program or in functions and can be updated or re-declared. Let variables can be updated but not re-declared. They are accessible within its block and initialzed with undefined. Const cannot be re-declared or updated and are block-scoped. A name is followed after the keyword to declare the variable. Let and const are more commonly used as var opens up security vulnerabilities. Semicolons are used at the end of each line in JS to signify end of statement.

Variables can be assigned a value using the = assignment operator.

var Hamburger;

const monthOfBirth = 'June';

let myVar = true;

Operators

JavaScript has built-in operators. Operators are symbols between values that perform operations.

Arithmetic Operators

Arithmetic operators include + addition, - subtraction, * multiplication, and / division. Another useful operator is the % remainder operator which returns the remainder of a quotient. Examples with descriptions are listed below.

let addition = 1 + 1; // 2

let subtraction = 2 - 3; // -1

let multiplication = 4 * 5; // 20

let division = 12 / 2; // 6

const remainder = 4 % 3 // 1

a * (b + c) // grouping

person.age // member

person[age] // member

!(a == b) // logical not

a != b // not equal

typeof a // type (number, object, function...)

x << 2 x>> 3 // minary shifting

a = b // assignment

a == b // equals

a != b // unequal

a === b // strict equal

a !== b // strict unequal

a < b a> b // less and greater than

a <= b a>= b // less or equal, greater or eq

a += b // a = a + b (works with - * %...)

Conditionals

Conditionals determine whether code will run or not. They control the program. There are three basic types of conditionals in JavaScript which are "If" statements, "Else" statements, and "Else if" statements.

  • "If" statements test whether a statement is true and then executes code
  • "Else" statements are for the same condition that proves false, followed by code execution
  • "Else-if" selects a new test if first condtion is proven false.

If Statement

if (2 > 1) {
    *code to execute*
}

Else Statement

if (2 > 1) {
    *code to execute*
} else {
    *code to execute when if is false*
}

Else-if Statement

if (false) {
    *code to execute*
} else if (true) {
    *code to execute when statement is true*
} else {
    *code to execute*
}

Loops

Loops offer a way to repeat code.

For Loops

A for loop executes until a specified condition proves false.

for ([initialExpression]; [conditionExpression]; [incrementExpression]) statement

Do...While Loops

Do while loops execute repeatedly until certain condition proves false. One statement is always executed before checking of condition. before running the while loop.

do
  statement
while (condition);

While Loops

While loops execute statements after finding a condition true. The condition is tested before the statement is run.

while (condition)
    statement

Labeled

Label provides loops with an identifier so that it can be referred to later on.

label :
    statement

johnLoop:
while ( theJohn === true) {
    doSomething();
}

Break

The break statement is used to stop a loop. It can be used with labeled statements as well.

break;
break [label];

Continue

Continue is used to restart a while, do-while, for, or label loop. It end the current iteration of the loop and continues execution of the loop with its next iteration.

continue [label]; let i = 0; let n = 0; while (i < 5) { i++; if (i === 3) { continue; // when i becomes 3 this continue statement causes the loop to reset back to i++ and become 4 } n += i; console.log(n); } //1,3,7,12 let i = 0; let n = 0; while (i < 5) { i++; if (i === 3) { // continue; } n += i; console.log(n); } // 1,3,6,10,15

For...in

The for...in statement is used to iterate over enumerable properties in an object.

const object = { x: 1, y: 2, z: 3 }; for (const property in object) console.log(`${property}: ${object[property]}`) } // "x: 1" // "y: 2" // "z: 3" <

For...of

The for...of loop iterates through all values of iterable objects such as Arrays, Strings, Maps, and NodeLists.


const array = ['0', '1', '2']

for (const element of array) {
    console.log(element)
}

// "0"
// "1"
// "2"

Functions

A function is a block of code designed to complete a specified task. Functions can take zero or more parameters. The body of the function can take as many statements as necessary. Variables are allowed to be declared inside of functions. Functions can return values using the return keyword. If no return statement is issued a function will return undefined.


function add(x, y) {
    var total = x + y;
    return total;
}
Arrays

Arrays are lists of items. Each item in an array is called an element.

const things = ["watch", "book", "phone"]

Indexes

Each string in the things array is its own element and have indexes, starting with 0.The "watch" element in this array is at index 0 and can be called as follows: 
things[0] // watch

Mutability

Elements in an array are mutable, meaning that they can be changed.

things[0] = "ring" console.log(things) // ['ring', 'book', 'phone'] A helpful property to determine the amount of elements in an array is .length. 
console.log(things.length) // 3
Objects

Objects are a data type in JavaScript that is used to store keyed collections. They are preferred over arrays when the data to be stored is more complex.

const character1 = {
    Name: "Morthul",
    Class: "Warlock"
    Specialization: Destruction,
    Level: 50,
    War Mode: True
}
Key-Value Pairs

Each key in an object has a respective vaslue. The data before the : is the key. Data after the : is the value. To call a specific value in an object you can use either dot or bracket notation. Bracket notation is best for when keys have spaces as dot notation will not work.


character1.Name // Morthul (using dot notation) character1['War Mode'] // true (using bracket notation)
